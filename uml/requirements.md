# ✅ REQUIREMENTS.md: Требования к приложению TaskManager

## 1. Функциональные требования (FR)
| ID | Требование | Описание |
|:---:|:---|:---|
| FR-1 | **Добавление задачи** | Пользователь должен иметь возможность добавить новую задачу, указав название и описание. Приложение автоматически присваивает ID и статус "новая". |
| FR-2 | **Просмотр списка** | Приложение должно отображать список всех задач с их ID, названием, описанием и текущим статусом. |
| FR-3 | **Изменение статуса** | Пользователь должен иметь возможность изменить статус задачи на "в работе", "завершена" или "новая". |
| FR-4 | **Редактирование** | Пользователь должен иметь возможность изменить название и/или описание существующей задачи по ее ID. |
| FR-5 | **Удаление (одиночное)** | Пользователь должен иметь возможность удалить конкретную задачу по ее ID. |
| FR-6 | **Удаление (массовое)** | Пользователь должен иметь возможность удалить сразу несколько задач (например, все завершенные). |
| FR-7 | **Автосохранение** | При любом изменении данных (добавление, редактирование, удаление) или выходе из приложения данные должны быть сохранены в файл. |

## 2. Нефункциональные требования (NFR)
* **NFR-1 (Хранение):** Данные о задачах должны храниться в файле `tasks.json` в формате JSON.
* **NFR-2 (Устойчивость):** Приложение не должно аварийно завершать работу (падать) при неверном вводе пользователя (например, при вводе буквы вместо ID).
* **NFR-3 (Интерфейс):** Приложение должно работать в консольном режиме.

## 3. User Stories
* **US-1:** Как пользователь, я хочу **добавить задачу** «Сдать проект», чтобы **не забыть дедлайн**.
* **US-2:** Как пользователь, я хочу **видеть статус задач**, чтобы **понимать прогресс** и приоритезировать работу.
* **US-3:** Как пользователь, я хочу **удалить завершенные задачи**, чтобы **очистить список** от неактуальных дел.
